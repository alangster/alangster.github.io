Enumerables


If you are like me (and I pity you if that is the case), you probably do not quite know what enumerable means. So, here it is: Enumerable means countable. It’s really that simple. So, in Ruby, when we say “enumerable,” we are talking about methods that operate on objects that are countable. Such objects are arrays and hashes, and when we use an enumerable, we iterate over each element of the array or hash, and do something with or to each element. Of course, this can be achieved with a “for” loop, but using enumerables is more efficient, more direct, and just plain sexy.


One of the coolest enumerable methods is the .map method. The .map method iterates over each element of the array and performs whatever task you give it on each element. The output is then saved to a new array. For example, if you had an array of strings of the titles of all the Harry Potter novels (call it harry_potters), and you wanted to add “by JK Rowling” to the end of each title (because you worry some people led deprived childhoods), you could simply call harry_potters.map { |book| book + “by JK Rowling” }. It’s that simple. The resulting array would be separate from the original array of titles, unless of course you decided to call .map! Calling .map! permanently changes the original array.
